# AI-Generated Meta-Linux OS Specification

This repository defines a Linux distribution where all userland components,
init system, package manager, installer, configuration files, desktop
environment, compatibility layers, language tooling, and documentation are
generated by AI models inside VS Code.

The human role is limited to providing structure, reviewing changes, and
running build scripts. All logic, code, and system behavior should be
AI-generated and AI-maintained, except for explicitly fetched third-party
components.

---

## 1. Project Goals

- Fully bootable Linux OS generated and maintained by AI.
- Usable by non-technical users.
- Customizable look, feel, themes, and desktop environment (smooth, rounded UI).
- Minimal, modular, clean architecture with rollback support.
- No picky partitioning or complex installation steps.
- Support for modern hardware (NVIDIA, AMD, Intel) and Apple hardware (where possible).
- Support for legacy Windows software via Wine, Proton, and a dedicated Windows VM.
- Support for Android apps via Waydroid + microG.
- Support for Chrome/Chromium-based browsers.
- Support for normal VMs (QEMU, VirtualBox optional).
- Support for cross-distro tools (Kali, Fedora, Debian/Ubuntu/Mint, openSUSE, MX).
- Support for a Universal Package Compatibility Layer (UPCL).
- Support for a Language Runtime Manager (langmgr) for modern, legacy, and esoteric languages.
- Built-in privacy and security using established open-source tools.
- Browser isolation and specialized lightweight VMs for sensitive tasks.

---

## 2. Non-AI Components (Fetched or Installed)

These components must NOT be generated by AI. They are fetched or installed
as-is to ensure hardware compatibility, security, and stability.

Fetched/installed by scripts in `build/`:

- Linux kernel (from kernel.org)
- musl libc
- TinyCC (tcc) and/or Clang/LLVM toolchain
- BusyBox or ToyBox (optional fallback utilities)
- GRUB or Syslinux bootloader
- QEMU (for VMs, including Windows Legacy Mode)
- Waydroid (Android container)
- microG (open-source Play Services replacement)
- Wine
- Proton (from Steam or Proton-GE where possible)
- DOSBox (for retro languages/tools)
- AppArmor or SELinux
- Firejail or Bubblewrap
- Flatpak
- uBlock Origin (for supported browsers)
- Btrfs tools (for snapshot/rollback)
- Optional: VirtualBox, Nix, Snapd

The AI may generate wrappers, configs, policies, and integration scripts for
these, but not the components themselves.

## 3. AI-Generated Components

### Core System
- init system (`/sbin/init`)
- service manager
- shell (`aish`)
- core utilities (toy versions of ls, cp, mv, cat, etc.)
- package manager (`aipkg`)
- Universal Package Compatibility Layer (UPCL)
- configuration system
- logging system
- system defaults
- rollback integration (Btrfs snapshots, Snapper-like tool)

### Desktop Environment
- window manager
- panel / dock
- launcher
- settings app
- theme engine
- theme packs
- wallpapers, icons, branding
- smooth, rounded UI (no sharp right angles)

### Installer
- auto-partitioning
- auto-formatting
- auto-installation
- bootloader setup
- user creation
- hardware detection
- driver auto-install (NVIDIA/AMD/Intel/Wi-Fi/Bluetooth/etc.)

### Compatibility Layers
- UPCL for DEB, RPM, PKGBUILD, AppImage, Flatpak, Snap (optional), Nix (optional)
- Kali tools integration
- Fedora/openSUSE tools integration
- MX Tools integration
- NVIDIA driver integration scripts
- AMD/Intel GPU setup scripts
- Waydroid + microG integration
- Wine + Proton integration
- Windows Legacy Mode VM wrapper
- VM templates and launchers (banking, suspicious files, dev, browser)

### Language Runtime Manager (langmgr)
- language registry (modern, legacy, esoteric, retro, VM-based)
- `lang install <language>` installer
- `run <file>` universal runner
- integration with DOSBox, Wine, Windows VM
- modular, extensible design

### Privacy & Security Integration
- AppArmor/SELinux policies
- Firejail/Bubblewrap profiles
- Flatpak sandboxing defaults
- encrypted home (LUKS/fscrypt) integration
- firewall configuration
- DNS-level ad/tracker blocking
- browser isolation (sandbox/micro-VM)
- universal uninstaller for system, sandboxed, and VM-based apps

### Build System
- kernel build wrapper
- userland build scripts
- rootfs assembly
- ISO creation pipeline
- QEMU testing scripts

### Documentation
- man pages
- help system
- user guide
- developer guide
- troubleshooting guide
